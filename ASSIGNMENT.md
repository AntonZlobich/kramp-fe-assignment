# Kramp Frontend Assignment

Welcome! This assignment is designed to evaluate your ability to review, analyze, and improve a real-world codebase.

## Objective
You will review an existing Next.js e-commerce application that is fully functional but contains various issues related to code quality, performance, accessibility, framework usage, and best practices.

**Your goal is to:**

- Analyze the codebase critically
- Identify issues across multiple dimensions
- Prioritize improvements based on impact
- Implement as many meaningful improvements as possible

We are primarily interested in how you think and communicate, not just how many issues you fix.

## The Application

This is a simple webshop with the following features:
- Homepage with featured products
- Search functionality (in the header)
- Product detail pages
- Shopping cart
- Checkout page

## What We're Looking For

1. **How you approach issues**: Can you systematically identify problems across different areas?

2. **How you prioritize**: Can you distinguish between critical issues that affect users and minor improvements? What would you fix first and why?

3. **Technical depth**: Do you understand modern React patterns, Next.js data fetching strategies, web performance, accessibility standards, and TypeScript best practices?

4. **Communication**: Can you clearly explain what's wrong, why it matters, and how to fix it?

## Your Deliverable

### 1. **Issue Report**
A document listing the issues you found, including:
- **Description:** What is the problem?
- **Impact:** How does this affect users or other aspects?
- **Priority:** High / Medium / Low

> [!IMPORTANT]
> Do not commit the Issue Report file. Please attach it to your submission email.

### 2. **Code Fixes**
Implement fixes for as many issues as you can within the time limit. If you don't have enough time to implement a fix, we value solution proposals, please add your proposed approach to the [Issue Report](#1-issue-report) file.

Commit your changes with clear, descriptive commit messages.

## Submission

1. **Fork this repository** to your GitHub account
2. Make your changes in your fork
3. **Send us the link** to your forked repository when it's ready
4. **Attach your Issue Report** in the email

## Areas to Consider

While reviewing the code, consider looking at:
- TypeScript usage and type safety
- Next.JS/React APIs
- Performance
- Accessibility
- Code organization, maintainability and security
- Tooling and environment setup

> [!NOTE]
> The application should remain fully functional after your changes.

## Out of Scope

- Do not change the framework (must remain Next.js + React)
- Do not migrate to the App Router or React Server Components
- Avoid major structural changes in [./apps/server](./apps/server/)
  - Minor fixes or improvement proposals are welcome
  - The schema intentionally has some timeouts to simulate slow APIs, please do not remove them to make it fast.

## Time Expectations

You are expected to spend approximately 4 hours on this assignment. You don't need to fix everything. Focus on prioritizing the most important issues.

## Questions?

If something is unclear, make an assumptions and document them in your [Issue Report](#1-issue-report).

Good luck! We're excited to see your analysis and approach.
